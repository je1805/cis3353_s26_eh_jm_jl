# Security Cleanup Guide

## Overview

This repo previously had secrets (passwords, SSL private keys, editor junk) committed
directly into git history. Even though they've been removed from the current commit,
**anyone who clones the repo can still see them in older commits**.

This guide walks through the full cleanup: scrubbing history, rotating secrets, and
keeping things clean going forward.

## Files You Should NEVER Commit

These are now in `.gitignore`, but here's the full list so everyone on the team knows:

### Secrets & Credentials
- `.env` — contains all passwords; use `.env.example` as the template
- `configs/ssl/` — entire folder; generated by `scripts/setup/generate-certs.sh`
- `*.pem` — any certificate or private key file
- `*.key` — any private key file
- `secrets/` — reserved folder for any future secret storage

### macOS Junk (from SharePoint/zip downloads)
- `__MACOSX/` — invisible metadata folders created when extracting zips on Mac
- `.DS_Store` — Finder metadata; leaks folder structure info
- `.AppleDouble` — resource fork data

### Editor Artifacts
- `*.swp`, `*.swo` — Vim swap files (often contain partial file contents)
- `*~` — Emacs backup files

### Other
- `.env.backup.*` — backup files created by `rotate-secrets.sh`
- `*.csr`, `*.srl` — intermediate cert files from cert generation

## Step-by-Step Cleanup

### Prerequisites

Install `git-filter-repo` (needed for Step 2):

```bash
# macOS
brew install git-filter-repo

# Linux (Ubuntu/Debian)
sudo apt install git-filter-repo

# Any platform with pip
pip install git-filter-repo
```

### Step 1: Make Sure Your Working Tree is Clean

```bash
cd /path/to/cis3353_s26_eh_jm_jl
git status
```

If there are uncommitted changes, commit or stash them first.

### Step 2: Scrub Secrets from Git History

This removes `.env`, `*.pem`, `.DS_Store`, `__MACOSX/`, and `*.swp` from **every
past commit**. It rewrites history, so there's no undo (other than the backup branch
the script creates).

```bash
bash scripts/setup/scrub-history.sh
```

The script will:
1. Ask you to confirm (type `yes`)
2. Create a backup branch (just in case)
3. Remove all secret files from every commit in history
4. Garbage-collect the old objects so they're truly gone

### Step 3: Rotate All Secrets

The old passwords are still *known* even after scrubbing history — anyone who saw
the repo before could have copied them. Generate fresh ones:

```bash
bash scripts/setup/rotate-secrets.sh
```

This will:
1. Back up your current `.env`
2. Generate 5 new random 20-character passwords
3. Write a fresh `.env` with the new passwords
4. Regenerate all SSL certificates

The script prints your new Wazuh Dashboard password at the end — save it somewhere.

### Step 4: Re-add the Remote and Force Push

`git-filter-repo` removes the remote origin as a safety measure. Add it back and
force-push:

```bash
git remote add origin <your-github-repo-url>
git push --force --set-upstream origin main
```

### Step 5: Tell Your Teammates

**Everyone who has cloned this repo must delete their local copy and re-clone.**
A regular `git pull` will NOT work after history has been rewritten — it will create
a mess of merge conflicts.

```bash
# Teammates should run:
rm -rf cis3353_s26_eh_jm_jl
git clone <your-github-repo-url>
cd cis3353_s26_eh_jm_jl
cp .env.example .env
# Then edit .env with the new passwords (shared securely, not over GitHub)
bash scripts/setup/lab-setup.sh
```

### Step 6: Set Up the Lab Again

After rotating secrets, the running containers (if any) will have stale credentials.
Rebuild:

```bash
docker compose down
docker compose build --no-cache
docker compose up -d
```

## Ongoing Best Practices

1. **Never put passwords in code.** Use `.env` for secrets, `.env.example` for the
   template. Share real passwords via a secure channel (not GitHub, not email).

2. **Check before committing.** Run `git diff --cached` to review staged changes.
   If you see a `.env` or `.pem` file, unstage it: `git reset HEAD <file>`

3. **If you accidentally commit a secret**, don't just delete it in the next commit.
   The old commit still has it. Re-run `scrub-history.sh` immediately.

4. **Regenerate certs per-machine.** Each person setting up the lab should run
   `scripts/setup/generate-certs.sh` or `scripts/setup/rotate-secrets.sh` locally.
   Don't share cert files.
